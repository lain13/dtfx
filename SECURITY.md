# セキュリティポリシー

## サポート範囲

セキュリティ修正は、既定ブランチの最新コードへ優先的に適用します。正式なリリースポリシーが定まるまでは、過去バージョンの個別サポートを保証しません。

## 脆弱性の報告

公開 Issue に認証情報、攻撃コード、機密性のある運用情報を投稿しないでください。リポジトリ所有者の非公開報告チャネルである GitHub Security Advisories を利用してください。受領後、再現状況と対応方針を可能な限り速やかに返信します。

## 運用上の注意

- DTFX のジョブ XML は SQL と Windows コマンドを実行できるため、コードと同等に保護してください。
- 信頼できない XML や config ファイルを実行しないでください。
- データベースアカウントには、ジョブに必要な最小限の権限だけを付与してください。
- 接続文字列やパスワードをリポジトリへコミットしないでください。
- ジョブ定義、設定ファイル、入力ファイルを変更できる利用者を制限してください。
- `ZipArchive` のパスワード機能を、強力な秘密情報保護や暗号化ストレージの代替として使用しないでください。
