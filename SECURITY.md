# Security Policy

## 지원 범위

보안 수정은 기본 브랜치의 최신 코드에 우선 적용합니다. 정식 릴리스 정책이 마련되기 전까지 과거 버전에 대한 별도 지원은 보장하지 않습니다.

## 취약점 제보

공개 이슈에 자격 증명, 공격 코드 또는 민감한 운영 정보를 올리지 마세요. 저장소 소유자의 비공개 보안 제보 채널(GitHub Security Advisories)을 사용해 주세요. 접수 후 재현 여부와 대응 계획을 가능한 한 빠르게 회신합니다.

## 운영 시 주의 사항

- DTFX 작업 XML은 SQL과 Windows 명령을 실행할 수 있으므로 코드와 동일하게 보호해야 합니다.
- 신뢰할 수 없는 XML이나 config 파일을 실행하지 마세요.
- DB 계정에는 작업에 필요한 최소 권한만 부여하세요.
- 연결 문자열과 암호를 저장소에 커밋하지 마세요.
- ZIP 암호 기능을 강력한 비밀 보호나 암호화 저장소의 대체 수단으로 사용하지 마세요.
